Insertion sort

Insertion sort is also a very simple sorting algorithm. This algorithm works on the principle of inserting elements in a particular position hence the name insertion sort. Think about how you sort a pack of cards, you just take a part of pack considering it as sorted and the other half pack as unsorted and picks every card and insert it at its correct position. Same is with the insertion sort.

Insertion sort Algorithm


			INSERTION – SORT(A)
	For i = 1 to n
		key  A[i]
		j  i – 1
		while j > = 0 and A[j] > key
			A[j+1]  A[j]
			j  j – 1
		End while
		A[j+1]  key
	End for

To sort an array of size n in ascending order:

  • Iterate from arr[1] to arr[n] over the array.
  • Compare the current element (key) to its predecessor.
  • If the key element is smaller than its predecessor, compare it to the elements before. Move the greater elements one position up to make space for the swapped element.

    • As you can see in the above figure, The array elements are compared with each other sequentially and then arranged simultaneously in some particular order.

void insertionSort(int arr[], int length) 
{
    int i, j, key;
    for (i = 1; i < length; i++) 
    {
        key = arr[i];
        j = i-1;
        
        while (j >= 0 && arr[j] >key) 
        {
            arr[j+1] = arr[j];
            j--;
        }
        arr[j +1] = key;
        
    }
    cout << "Sorted Array: ";
    // print the sorted array
    printArray(arr, length);
}

// function to print the given array 
void printArray(int array[], int size)
{ 
    int j;
    for (j = 0; j < size; j++)
    {
        cout <<" "<< array[j];  
    }
    cout << endl;
}

    Applications:
    This sorting algorithm can be used on arrays having at most 30 elements. Not more than that as it is not appropriate to use this algorithm for huge data.

    Complexity Analysis:

    Time Complexity

  • Worst Case Time Complexity [ Big-O ]: O(n2)
  • Best Case Time Complexity [Big-omega]: O(n)
  • Average Time Complexity [Big-theta]: O(n2)
